## What I Learned

- How to read `ip a` output — interfaces, IPs, subnets, MAC prefixes
- VMware NAT network layout and what each infrastructure host does
- nmap basics: ping sweep vs service scan, what `-sV` and `-O` do
- How services advertise their version in banners — and why that's a security problem
- The vsftpd 2.3.4 backdoor: what a supply chain attack is, how the smiley-face trigger works
- Why privilege inheritance matters — root service = root shell
- Port 6200 is ephemeral — only exists while the backdoor session is active
- Persistence: why a shell session isn't enough for a real attacker

---

## Questions / Observations

- If vsftpd had run as a non-root user, the exploit would have landed with limited privileges
- Port 6200 wasn't visible in nmap because it didn't exist yet — dynamic ports won't show in a pre-trigger scan
- Real servers in 2011 ran this software publicly, announced their version freely, and were wide open to this exact attack from anywhere on the internet
