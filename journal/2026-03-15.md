## Questions / Observations

- If vsftpd had run as a non-root user, the exploit would have landed with limited privileges
- Port 6200 wasn't visible in nmap because it didn't exist yet — dynamic ports won't show in a pre-trigger scan
- Real servers in 2011 ran this software publicly, announced their version freely, and were wide open to this exact attack from anywhere on the internet

### Starlink router scan (192.168.1.1)
- Port 22: SSH (OpenSSH 8.4) — Starlink remote management
- Port 80: HTTP (Golang) — API endpoint, not a browser UI (returns 500)
- Port 9000-9003: gRPC/HTTP — Starlink app API
- No traditional admin panel — locked down by design

### Key lessons
- iPhone blocks ping but has open ports — `-Pn` bypasses ping check
- Open port ≠ accessible service (Starlink HTTP returns 500, requires specific client)
- Modern locked-down devices vs Metasploitable: night and day difference in attack surface
